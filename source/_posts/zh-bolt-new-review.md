# Bolt.new 评测：用描述来构建 Web 应用

**一句话评价：** Bolt.new 是从想法到可运行原型的最快方式。但别指望它产出生产级代码。

## Bolt.new 是什么？

Bolt.new 是 StackBlitz 出品的浏览器端 AI 应用构建器。用英文描述你想要什么，它就能生成一个全栈 Web 应用——前端、后端、数据库全包。完全在浏览器里通过 WebContainers 运行，不需要安装任何东西。

口号是"和 AI 聊天，拿到应用"。对于有想法但不想花几周写样板代码的一人公司，听起来像魔法。

## 我怎么用的

试着做了三个东西：
1. 一个带等待名单注册的简单落地页
2. 一个展示模拟数据带图表的仪表盘
3. 一个跟踪自由职业发票的基础 CRUD 应用

落地页大概 5 分钟搞定，开箱即用，看起来很专业。仪表盘需要调整几次提示词，15 分钟完成。发票追踪器花了 30 分钟，中间修了几个 bug 加了一些功能。

工作流程：描述需求→预览结果→描述问题→重复。就像和一个写代码很快但需要非常具体指令的人结对编程。

## 优点

- **做原型快得离谱。** 从"我有个想法"到"这是个能用的 demo"只要几分钟，这对验证想法来说是颠覆性的。
- **零配置。** 全在浏览器里跑。没有 Node.js 版本问题，没有依赖地狱，打开标签页就开始。
- **开箱全栈。** 不只是 HTML，前端、后端、基础数据库都能处理，拿到的是一个真正的应用。
- **代码可导出。** 代码是你的。下载下来，部署到别处，用自己的编辑器继续开发。没有锁定。
- **不懂代码也能用。** 能写清楚英文描述就能搭东西，门槛极低。

## 缺点

- **代码质量堪忧。** 生成的代码能跑但很乱。大量内联样式、不一致的模式、极少的错误处理。原型够用，维护起来很痛苦。
- **复杂逻辑容易翻车。** 需求越具体，Bolt 越容易懵。状态管理、复杂表单、边界情况都需要手动修。
- **没有版本控制集成。** 能导出代码但工具本身没有 Git 集成。想迭代就在 Bolt 编辑器里做。
- **费用可能超预期。** 复杂项目迭代几次 token 消耗很快，可能不知不觉就用完月度额度。

## 定价

| 免费版 | $0 | 每日有限 token |
| Pro | $20/月 | 更多 token，优先访问 |
| Team | $40/月/人 | 团队功能 |

一人开发者先用免费版试水。如果经常做原型验证，Pro 版 $20/月值——花一个下午验证一个商业想法，比花一个月强。

## 适合谁

需要快速验证想法的一人公司，Bolt.new 是神器。一个下午做出 MVP，给潜在客户看，拿反馈，迭代。原型速度无敌。

设计师也适合，能做出能用的功能原型，不只是静态设计稿。

不推荐用来构建长期维护的生产应用。代码需要大量清理。用 Bolt 做原型，想法验证通过后正经重写。

## 替代方案

- **Lovable：** 类似概念，UI 方法略有不同。两个都试试，看哪个更懂你的提示词。
- **v0 by Vercel：** 专注 UI 组件，不那么全栈但输出更干净。主要需要前端的话挺好。
- **Replit Agent：** 更偏开发者，代码质量更好，但学习曲线更陡。
