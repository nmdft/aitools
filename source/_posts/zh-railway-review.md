如果你曾经花一下午的时间跟 AWS 的 IAM 策略、Nginx 配置、或者 Docker Compose 文件搏斗，只为把一个简单的 web 应用上线——你应该理解那种痛苦。Railway 的存在就是为了解决这个问题。

Railway 是一个平台即服务（PaaS），它从你的 GitHub 仓库、Docker 镜像、甚至本地目录拉取代码，用最少的配置让它在生产环境跑起来。你可以把它理解为现代版的 Heroku，但定价更合理，内置数据库，而且仪表盘做得真心不错。

我用 Railway 好几个月了，从 side project API 到带 Postgres 后端的全栈应用都部署过。以下是我的真实体验。

![Railway 首页](/images/railway-homepage.png)

## Railway 到底能干什么

核心卖点很简单：连接 GitHub 仓库，剩下的 Railway 搞定。它自动检测你的框架（Next.js、Express、Fastify、Django、Rails 等等），配置构建流水线，分配域名，应用就上线了。不需要 Dockerfile，当然你想用也行。

但 Railway 不只是"部署一个 web 应用"那么简单：

- **数据库**：一键部署 Postgres、Redis、MySQL、MongoDB。它们作为项目里的一等公民服务运行，有自动备份，跟你的应用走私有网络通信。
- **定时任务**：原生支持 cron job，不需要额外接 cron-job.org 之类的第三方。
- **Worker 和后台进程**：跟 web 服务器放在同一个项目里跑。
- **预览环境**：每个 PR 自动获得一个独立部署 + 独立 URL，团队成员可以先测试再合并。
- **私有网络**：服务之间走内网通信，数据库不暴露到公网。
- **模板市场**：一键部署整个技术栈——博客、监控面板、聊天应用，社区模板点一下就上。

跟 AWS/Azure/GCP 的区别在于开发者体验。仪表盘把实时构建日志、部署日志、CPU/内存指标、HTTP 日志全放在一个页面里。回滚到上一个版本只需点一下。环境变量按环境（生产、预发布）管理，不用再去翻那个忘了更新的 `.env` 文件。

## AI Agent 集成

Railway 最近在 AI Agent 方向投入很大，而且不是喊口号：

- **MCP Server**：官方提供了 MCP（Model Context Protocol）服务器，AI 编程助手（比如 Claude Code）可以直接操作你的 Railway 部署——查部署状态、管理服务、看日志。
- **Agent Skills**：AI Agent 可以通过 API 部署、配置、管理 Railway 资源。
- **Railway Metal**：自建裸金属基础设施，专门为 AI 工作负载设计。

对一人公司做 AI 产品的人来说，这个愿景很吸引人：你对编程 agent 说"把这个部署到 Railway"，然后就搞定了。

## 定价：按秒计费

Railway 的定价是真正的按量付费。没有预留实例，没有"选个套餐然后祈祷够用"。你的服务实际消耗多少 CPU 和内存，按秒收费。

| 套餐 | 月费 | 说明 |
|------|------|------|
| **Free** | $0 | 30天试用送 $5 额度，之后有资源限制 |
| **Hobby** | $5 起步 | 含 $5 使用额度，最高 48 vCPU / 48 GB |
| **Pro** | $20 起步 | 含 $20 使用额度，最高 1 TB，无限席位 |
| **Enterprise** | 定制 | SSO、RBAC、HIPAA、专属虚拟机 |

每秒费率：
- CPU：$0.00000772 / vCPU / 秒
- 内存：$0.00000386 / GB / 秒
- 存储卷：$0.00000006 / GB / 秒
- 出站流量：$0.05 / GB

一个典型的 Next.js 应用 + Postgres，用 Hobby 套餐大概 $5-10/月。比很多 VPS 便宜，而且你有自动扩缩、预览部署、正经的仪表盘——不用 SSH 到一台服务器上折腾。

## 我喜欢的地方

**部署速度真的快。** 我计过时，一个普通的 Node.js 应用从 push 到上线不超过 2 分钟。预览环境起来更快。对比一下 AWS——等 CodePipeline 发现你的提交、等 ECS 拉镜像、等 ALB 健康检查通过——半小时没了。

**数据库体验省心。** 启动一个 Postgres 实例只需点一下。自动生成私有 URL、自动备份、你的应用通过内网连上去。不用配 RDS 参数组、不用搞 VPC 子网、不用设安全组规则。

**回滚真的好用。** 仪表盘里点一下就回到上一个版本。不用去 ECR 里翻旧 Docker 镜像的 tag。

**CLI 不多余。** `railway up` 从本地目录部署。`railway logs` 看日志。`railway link` 把本地项目关联到 Railway 服务。该有的都有，不画蛇添足。

**模板被低估了。** 需要 Metabase？点一下部署。需要 Umami 分析？点一下部署。需要 Plausible + Postgres 的完整分析栈？有现成模板。光这一项就省了几个小时的搭建时间。

## 我不喜欢的地方

**试用期过了之后免费套餐很抠。** 30 天送 $5 额度，然后你只剩 1 个项目、3 个服务，资源限制很紧。作为一个"永久免费"套餐，基本只能跑个 hello-world。如果你真的预算为零，一台 $5/月的 Hetzner VPS 给你更多。

**冷启动偶尔烦人。** 没有流量的服务会被缩到零，首次请求需要冷启动，延迟 2-5 秒。可以在设置里关掉，但费用会增加。

**不能 SSH 进容器。** 如果出了问题需要进运行中的容器里排查，你只能看日志。大多数情况下日志够用了，但偶尔真的需要 shell 进去看看。

**出站流量不便宜。** $0.05/GB，如果你的服务有大量外发流量（传图片、媒体资源），费用会涨得很快。存静态资源用 Cloudflare R2 或 S3 更划算，Railway 的优势在计算。

**"智能云"的标签吹过头了。** Railway 是个很棒的 PaaS，但叫自己"一站式智能云提供商"听起来像它能做更多。它不会自动优化你的查询、不会帮你重写代码。它就是把你的东西部署好而已——这就够了，不需要过度包装。

## Railway vs. 竞品

| 特性 | Railway | Vercel | Render | Fly.io | Heroku |
|------|---------|--------|--------|--------|--------|
| 最适合 | 全栈应用 | 前端/JAMstack | 通用托管 | 低延迟场景 | 老项目 |
| 数据库 | 内置 | 靠集成 | 内置 | 靠扩展 | 插件（贵） |
| 定价 | 按量 | 按带宽 | 按量 | 按量 | 按 Dyno |
| Docker | 支持 | 有限 | 支持 | 擅长 | 支持 |
| 预览部署 | 有 | 有 | 有 | 有 | 有 |
| 免费层 | $5 试用 | 慷慨 | 有限 | 有限 | 已死 |

如果你在做 Next.js 前端，Vercel 可能仍然是最佳选择。如果你需要全球边缘计算和超低延迟，Fly.io 难以超越。但如果是"我有后端、有数据库、可能还有个 worker，想放在一个地方"这种场景，Railway 是最佳甜点。

## 谁适合用 Railway

**完美适合：**
- 独立开发者和小团队做 SaaS 产品
- 需要正经数据库的 side project（不只是静态页面）
- 厌倦了 Kubernetes、只想让应用跑起来的人
- 需要编程式部署的 AI Agent 工作流
- 快速原型——几分钟搭起全栈，不想要了就删掉

**不太适合：**
- 高流量应用（出站流量费用会很可观）
- 合规要求高的行业（SOC 2 有，但 HIPAA 要 Enterprise 套餐）
- 需要 SSH 进运行中容器的团队
- 想要长期免费托管的人

## 总结

Railway 就是 Heroku 本该进化成的样子。它把"开箱即用"的理念应用到现代工作负载——Docker 容器、多数据库、后台 worker、预览环境——而且定价对小项目很友好。

对一人公司来说，$5/月的 Hobby 套餐很难被打败。你得到的是一个有真实功能的正经平台，不是一个为了逼你升级而设计的阉割版免费套餐。AI Agent 集成是额外加分项，如果你已经在用编程助手的话。

它不是最便宜的选项（$5 Hetzner VPS 给你更多原始资源），也不是最强大的（AWS 能无限扩展）。但对于"我想把产品上线、不想操心基础设施"的人群——也就是我们大多数人——Railway 很好用。
